## Cuándo usar cada uno

- **Low Code**: cuando quieres validar una idea, ajustar instrucciones y comprobar viabilidad sin desarrollar todavía una extensión completa.
- **AL**: cuando el comportamiento ya está definido y necesitas seguridad, trazabilidad, despliegue controlado e integración con procesos reales.
